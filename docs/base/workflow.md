# 工作流是什么

工作流以流程图的形式，让你不用关心这么写代码的细节，让不懂代码的人也能用，只需通过文字描述和流程组合，就能生成可用代码。

### 为什么需要工作流（为啥不直接让 AI 写）

通过对话的方式让 AI 生成代码，容易出现逻辑混乱、结构不可控、对话过多容易迷失、与工程约束不一致的问题。

工作流的作用是：
把“业务意图”与“代码结构”拆开，让 AI 只负责局部逻辑，整体结构由流程统一控制。

### 工作流由哪些节点组成

##### 1. 开始 (也是工作流的入口)

该节点自动生成，无需手动添加，是整个工作流的入口，用于接收数据。

##### 2. 智能逻辑

用于将文字描述转换为业务代码。例如：处理某个数据，遍历数据，把数据赋给页面数据等等。

::: details 更多说明

为了让 AI 知道你要“操作谁”，否则它只知道你想干什么，不知道对谁干。

目标组件的用途：想要调用组件内部事件，就选择对应的组件。一般用来操作二维码组件和抽奖组件。

:::

##### 3. 页面跳转

选择目标页面，生成页面跳转代码。

##### 4. 网络接口

选择网络接口，生成对应的网络请求代码。

##### 5. 条件分支

根据不同条件执行不同的分支。例如：2>1 就执行符合条件的流程，否则就执行不符合的流程。

##### 6. 结束流程

用于结束流程整个工作流(不是结束某一个分支)，并且返回数据。

##### 7. 支付

<a href="https://uniapp.dcloud.net.cn/api/plugins/payment.html" target="_blank" rel="noopener noreferrer">方便调用微信小程序/支付宝支付接口。</a>

##### 8. 系统能力

用于使用 <a href="https://uniapp.dcloud.net.cn/api/ui/prompt.html" target="_blank" rel="noopener noreferrer">uniapp api</a>

比如：

弹提示

显示加载中

获取位置

打开相机

这些都属于系统能力。

### 工作流的使用技巧

快捷键
| 键组合 | 作用 |
| ---------------- | ------------ |
| Delete | 删除 |
| Ctrl + C | 复制 |
| Shift + 鼠标拖动 | 实现多选 |
| 鼠标双击连接线 | 删除连接线 |
